# Reference

## Repository layout

| Path | Purpose |
|---|---|
| `hooks/` | Active hook scripts installed into `~/.claude/hooks/`. |
| `hooks/_archive/learned-mistakes/` | Retired or reference scripts. Not installed. |
| `settings/hooks.fragment.json` | Generated hook wiring fragment for `~/.claude/settings.json`. |
| `data/redlines.tsv` | Default guard rules consumed by `hooks/guard.sh`. |
| `bin/build-fragment.sh` | Extracts source-machine hook wiring into the fragment file. |
| `install.sh` | Copies active hooks and optionally merges settings. |
| `uninstall.sh` | Restores newest settings backup and removes shipped files. |
| `test/` | Local shell test harnesses. |
| `contrib/one-offs/` | Incident-specific helpers that are not installed by default. |
| `docs/` | Design docs and user-facing documentation. |

## Lifecycle events

The pack targets five Claude Code hook lifecycle events.

| Event | Current coverage |
|---|---|
| `SessionStart` | Captures session metadata. |
| `SubagentStart` | Injects redline enforcement into subagents. |
| `UserPromptSubmit` | Adds context before the agent answers. |
| `PreToolUse` | Blocks or nudges risky tool calls. |
| `Stop` | Checks whether the assistant is trying to stop too early or defer work. |

`settings/hooks.fragment.json` is the wiring source for these events after it is
generated on the source machine.

## Installer behavior

`install.sh` accepts:

| Command | Behavior |
|---|---|
| `bash install.sh` | Copy hooks, back up settings, merge non-empty settings fragment. |
| `bash install.sh --dry-run` | Print actions without changing files. |
| `bash install.sh --no-settings` | Copy hooks and data only. |
| `bash install.sh --dry-run --no-settings` | Preview hook and data copies only. |

The installer copies top-level `hooks/*.sh` and `hooks/*.py`, excluding names
that match:

```text
*.retired
*.solution
*.archive.*
```

It also copies `data/redlines.tsv` to `~/.claude/hooks/redlines.tsv` and creates
`~/.claude/hooks/redlines.d/`.

## Settings merge safety

The merge step:

- reads `settings/hooks.fragment.json`
- counts hook entries across the five lifecycle events
- refuses to merge if the count is zero
- backs up the target settings file before writing
- overwrites only event keys with non-null payloads

This protects a target machine from replacing working local hook wiring with the
shipped all-null placeholder.

## Uninstaller behavior

`uninstall.sh`:

- restores the newest `~/.claude/settings.json.bak-*` backup when present
- removes every top-level hook file shipped by the current pack
- removes legacy archived and contrib files from older install layouts

It leaves unrelated user hook files alone.

## Verification commands

```bash
python3 -m json.tool settings/hooks.fragment.json >/dev/null
bash install.sh --dry-run --no-settings
bash test/straight-fix-no-ask.test.sh
```

Optional developer checks:

```bash
pre-commit run --all-files
shellcheck install.sh uninstall.sh bin/build-fragment.sh hooks/*.sh
```

## Generated files

`settings/hooks.fragment.json` should normally be generated by:

```bash
bash bin/build-fragment.sh > settings/hooks.fragment.json
```

The script keeps only recognized lifecycle events from the source machine's
`~/.claude/settings.json` hook block.
